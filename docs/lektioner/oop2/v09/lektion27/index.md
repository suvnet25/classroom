> Moment: `säkerhet`

## Innehåll

* Säkerhetsworkshop - Suverän Bank
    0. Vi går igenom de olika sårbarheter vi hittade igår
    1. Skapa issues i GitHub
        * Namnge issue så att det är tydligt vad det handlar om
        * Gärna med CWE-nummer i titel
        * Beskriv sårbarheten
        * Förslag på åtgärd
    2. Grupper om 2-3 personer väljer ut en sårbarhet att åtgärda
    3. Forka och klona repo
    4. Åtgärda sårbarheten
    5. Skapa pull request med beskrivning av vad ni har gjort. Länka till issue ni har åtgärdat.
    6. Code review
    7. Merge pull request
    * Vi tittar på hur applikationen uppdateras online
